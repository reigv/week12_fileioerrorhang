Function Pointers, Lambdas, File I/O, Error Handling

Straight-to-the-point notes on function pointers, lambda expressions, file I/O, and error handling, with simple examples and comments.

Quick idea

Function pointer = stores the address of a function.

Lambda = a short unnamed function you can write directly in code.

File I/O = reading from and writing to files.

Error handling = detecting problems and responding safely.

1) Function Pointers

A function pointer does not copy a function. It only stores the function's address, so your program can call that function later.

Think of it like this: a normal pointer stores the address of a variable. A function pointer stores the address of a function.

Basic syntax

// Pointer to a function with no parameters and no return value
void (*ptr)();

// Pointer to a function with parameters
int (*op)(int, int);

Example: basic function pointer

#include <iostream>
using namespace std;

void hello() {
    cout << "Hello from function\n";
}

int main() {
    void (*ptr)() = hello;   // ptr stores the address of hello
    ptr();                   // calls hello()

    return 0;
}
Important: ptr() calls the function stored in ptr. It does not restart the whole program and it does not loop by itself.

Why use function pointers?

  • To choose which function to run at runtime
  • To pass a function into another function
  • To build callback systems
  • To create tables or menus of functions

Example: function pointer with arguments

#include <iostream>
using namespace std;

int add(int a, int b) {
    return a + b;
}

int subtract(int a, int b) {
    return a - b;
}

int main() {
    int (*operation)(int, int); // pointer to a function that takes 2 ints

    operation = add;
    cout << operation(8, 3) << "\n";   // 11

    operation = subtract;
    cout << operation(8, 3) << "\n";   // 5

    return 0;
}

2) Array of Function Pointers

An array of function pointers lets you store multiple functions and choose one by index. This is useful for calculators, menus, and command systems.

#include <iostream>
using namespace std;

int add(int a, int b) { return a + b; }
int sub(int a, int b) { return a - b; }
int mul(int a, int b) { return a * b; }

int main() {
    // Array of pointers to functions
    int (*ops[3])(int, int) = { add, sub, mul };

    cout << ops[0](4, 2) << "\n";  // add(4, 2) = 6
    cout << ops[1](4, 2) << "\n";  // sub(4, 2) = 2
    cout << ops[2](4, 2) << "\n";  // mul(4, 2) = 8

    return 0;
}
This is like a list of function addresses. Each index points to a different function.

3) Callback Functions

A callback means: one function receives another function and calls it later.

#include <iostream>
using namespace std;

void done() {
    cout << "Task finished!\n";
}

void process(void (*callback)()) {
    cout << "Processing...\n";
    callback();   // call the function passed in
}

int main() {
    process(done);
    return 0;
}

What happens step by step?

  1. main() calls process(done)
  2. process() prints Processing...
  3. callback() actually means done()
  4. done() runs once
Common confusion: callback() does not make process() run again. It only runs the function stored in callback.

4) Lambda Expressions

A lambda is a short anonymous function. You usually use it when you need a function only once, especially with STL algorithms.

General syntax

[capture](parameters) -> return_type {
    // function body
};

Simple example

#include <iostream>
using namespace std;

int main() {
    auto greet = []() {
        cout << "Hello from lambda\n";
    };

    greet();
    return 0;
}

Lambda with parameters

#include <iostream>
using namespace std;

int main() {
    auto square = [](int x) {
        return x * x;
    };

    cout << square(5) << "\n";   // 25
    return 0;
}

Capture list

The capture list tells the lambda which outside variables it can use.

#include <iostream>
using namespace std;

int main() {
    int a = 10;
    int b = 20;

    auto show = [a, &b]() {
        // a is captured by value: lambda gets a copy
        // b is captured by reference: lambda can change the original b
        cout << "a = " << a << ", b = " << b << "\n";
    };

    show();
    return 0;
}
Capture Meaning
[] Capture nothing
[=] Capture everything by value
[&] Capture everything by reference
[x] Capture only x by value
[&x] Capture only x by reference

Lambda with STL

#include <iostream>
#include <vector>
#include <algorithm>
using namespace std;

int main() {
    vector<int> nums = {4, 1, 7, 2};

    sort(nums.begin(), nums.end(), [](int a, int b) {
        return a < b;   // sort in ascending order
    });

    for (int x : nums) {
        cout << x << " ";
    }
    return 0;
}
Use lambdas when the function is short and only needed in one place.

5) File I/O

File I/O means saving data to a file or reading data from a file.

Common classes

  • ofstream = output to file
  • ifstream = input from file
  • fstream = input and output

Need this header

#include <fstream>

Write to a file

#include <iostream>
#include <fstream>
using namespace std;

int main() {
    ofstream fout("data.txt");

    if (fout.is_open()) {
        fout << "Tom\n";
        fout << 95 << "\n";
        fout.close();
        cout << "File written successfully\n";
    }
    else {
        cout << "Could not open file for writing\n";
    }

    return 0;
}

Read from a file

#include <iostream>
#include <fstream>
#include <string>
using namespace std;

int main() {
    ifstream fin("data.txt");
    string name;
    int mark;

    if (fin.is_open()) {
        getline(fin, name);  // read a whole line
        fin >> mark;         // read an integer

        cout << "Name: " << name << "\n";
        cout << "Mark: " << mark << "\n";

        fin.close();
    }
    else {
        cout << "Could not open file for reading\n";
    }

    return 0;
}

Useful file functions

  • is_open() checks if the file opened properly
  • close() closes the file
  • getline() reads a full line
  • >> reads formatted data like int, double, string

6) Error Handling

Error handling means checking for problems and preventing the program from crashing or producing wrong results.

Type 1: Simple checking with if

#include <iostream>
using namespace std;

int divide(int a, int b) {
    if (b == 0) {
        cout << "Error: division by zero\n";
        return 0;
    }
    return a / b;
}

int main() {
    cout << divide(10, 2) << "\n";
    cout << divide(10, 0) << "\n";
    return 0;
}

Type 2: Exceptions

Exceptions are useful when an error is serious and you want to handle it separately.

#include <iostream>
#include <stdexcept>
using namespace std;

int divide(int a, int b) {
    if (b == 0) {
        throw runtime_error("division by zero");
    }
    return a / b;
}

int main() {
    try {
        cout << divide(10, 0) << "\n";
    }
    catch (const exception& e) {
        cout << "Caught error: " << e.what() << "\n";
    }

    return 0;
}

Keywords

  • try = code that might cause an error
  • throw = sends an exception
  • catch = handles the exception
In school exercises, first use simple if checks when possible. Use exceptions when your teacher expects more formal error handling.

8) What to Remember for Class

  • A function pointer stores an address, not a copy of the function.
  • callback() calls the passed function once. It does not repeat by itself.
  • An array of function pointers is useful when you want multiple selectable actions.
  • Lambdas are great for short code, especially with STL.
  • Always check if a file opened successfully.
  • Use if checks and exceptions to handle errors safely.
